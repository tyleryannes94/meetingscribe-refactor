# MeetingScribe

Local-first macOS app for recording, transcribing, and summarizing meetings — like Granola or Fathom, but everything stays on your machine. Includes a Whispr-Flow-style global dictation hotkey, a Note Transcriber for one-off voice notes, and an MCP server so Claude can query your meetings.

- **Audio**: ScreenCaptureKit for system audio + AVAudioEngine for mic — no virtual audio devices.
- **Transcription**: whisper.cpp running locally, in 5-minute rolling chunks during a call + a high-quality final pass on stop.
- **Summarization**: Ollama (default: `llama3.1:8b`) running locally — auto-started if not.
- **Calendar**: EventKit reads any calendar already in macOS Calendar (Google, iCloud, Outlook).
- **MCP**: Bundled `MeetingScribeMCP` server exposes 17 tools to Claude over JSON-RPC stdio — read **and** write (Claude can create tasks, add people, append memories, and write meeting notes back into the vault).

## Install (one command)

Requires **macOS 14 (Sonoma) or newer**.

```bash
git clone https://github.com/tyleryannes94/meetingscribe.git
cd meetingscribe
./install.sh
```

That single script:

1. Verifies macOS 14+ and Xcode Command Line Tools.
2. Installs Homebrew if missing.
3. `brew install whisper-cpp ollama jq`.
4. Downloads the default whisper model (`ggml-base.en.bin`, ~148 MB) and verifies its magic bytes.
5. Pulls the default Ollama model (`llama3.1:8b`, ~4.9 GB) and starts `ollama serve`.
6. Creates a self-signed code-signing certificate so TCC permissions survive rebuilds.
7. Builds the app + MCP server, signs them, installs to `/Applications`.
8. Launches MeetingScribe.

Two clicks you'll need to do yourself (macOS won't let scripts do these):

- **"Always Allow"** on the keychain dialog the first time codesign uses the new cert.
- Grant **Microphone**, **Screen Recording**, **Calendar (Full Access)**, **Notifications**, and (optionally) **Accessibility** when prompted. After granting Screen Recording, quit and relaunch the app.

## Architecture

```
 ┌─ Calendar (EventKit) ──┐
 │  Reads Google + iCloud │
 │  via macOS Calendar    │
 └─────────┬──────────────┘
           │ meeting metadata
           ▼
 ┌────────────────────────┐    ┌───────────────────────┐
 │  ScreenCaptureKit      │    │  AVAudioEngine        │
 │  system audio → m4a    │    │  microphone   → m4a   │
 │  (per 5-min segment)   │    │  (per 5-min segment)  │
 └──────────────┬─────────┘    └──────────┬────────────┘
                ▼                         ▼
        ┌────────────────────────────────────┐
        │ Every 5 min, whisper.cpp transcribes│
        │ the chunk → growing live transcript │
        └─────────────────┬──────────────────┘
                          ▼ (on stop)
        ┌────────────────────────────────────┐
        │ AVMutableComposition merges all    │
        │ segments → final whisper pass →    │
        │ Ollama summary → notes.md,         │
        │ summary.md, transcript.md          │
        └────────────────────────────────────┘
```

## What's inside the app

| Feature | Where |
|---|---|
| **Meeting recording** | Sidebar → Upcoming → 🎥 / record icon; or toolbar "Ad-hoc Recording" |
| **Live transcript (5-min chunks)** | Detail pane → Transcript tab during recording |
| **Add New Recording** | Detail pane of a past meeting (appends a segment) |
| **Transcribe Now** | Detail pane of a past meeting (re-merges + re-transcribes everything) |
| **My Notes** | Editable autosaving `notes.md` in detail pane |
| **AI Summary** | Auto-generated by Ollama; regenerated on Transcribe Now |
| **Tags** | Multi-select chip picker in every detail pane; presets: Skio, Recharge, Planning Calls, Product Team Calls, Slack Huddles, Impromptu Recording. Changing a tag moves the folder on disk |
| **Note Transcriber** | Top segmented control → Notes |
| **Whispr-Flow dictation** | F5 anywhere on macOS → speak → press F5 → text pasted at cursor |
| **Meeting-start notifications** | macOS notification 10s before each upcoming call with "Join & Record" |
| **Zoom/Meet auto-detect** | "Record impromptu" notification when a call starts that isn't on your calendar |
| **Audio playback** | Voice Memos-style player with ⏪15 / ⏯ / ⏩15 + scrub bar |

## MCP server (Claude integration)

The repo bundles a Model Context Protocol server at `MeetingScribe.app/Contents/MacOS/MeetingScribeMCP`.

**One-click setup**: Open the app → Settings → MCP Server → "Install in Claude Desktop". Restart Claude Desktop. Then ask Claude *"list my last 5 meetings"* or *"what were the action items from yesterday's planning call?"*.

**Tools exposed** (17 total — 12 read, 5 write):

_Read:_

| Tool | Args |
|---|---|
| `list_meetings` | `limit`, optional `tag` |
| `get_meeting` | `id` (returns metadata + transcript + notes + summary) |
| `get_transcript` | `id` |
| `get_notes` | `id` |
| `get_summary` | `id` |
| `list_voice_notes` | `limit` |
| `get_voice_note` | `id` |
| `list_people` | `limit` |
| `get_person` | `id` / name / email |
| `list_action_items` | optional `meetingID` |
| `get_action_item` | `id` |
| `search_everything` | `query` (people + meetings + action items, recency-boosted) |

_Write_ (patches raw JSON losslessly, append-only for notes, posts `vaultChanged`):

| Tool | Args |
|---|---|
| `create_action_item` | `title`, optional `dueDate`, `priority`, `meetingID` |
| `update_action_item` | `id`, fields to patch |
| `add_person` | `name`, optional `company`, `email`, `phone`, `role` |
| `add_memory` | person ref + `text` |
| `create_meeting_note` | `meetingID`, `text` (appended) |

**Manual install** (Claude Code, Cursor, etc.):

```bash
claude mcp add meetingscribe \
  /Applications/MeetingScribe.app/Contents/MacOS/MeetingScribeMCP
```

Or paste this into your client's MCP config:

```json
{
  "mcpServers": {
    "meetingscribe": {
      "command": "/Applications/MeetingScribe.app/Contents/MacOS/MeetingScribeMCP",
      "env": {
        "MEETINGSCRIBE_STORAGE": "/Users/YOU/Documents/MeetingNotes"
      }
    }
  }
}
```

## Settings reference

| Setting | Default | What it does |
|---|---|---|
| Storage folder | `~/Documents/MeetingNotes` | Where everything lives |
| Record microphone | ✓ | Capture your audio |
| Record system audio | ✓ | Capture the other side via ScreenCaptureKit |
| Auto-start when a calendar meeting begins | ✗ | One-minute polling for live events |
| Only show meetings with a conference link | ✓ | Hides time blocks and personal events |
| Notify at meeting start | ✓ | macOS notification with "Join & Record" |
| Detect Zoom / Meet impromptu calls | ✓ | Polls running apps; offers to record |
| Dictation hotkey | F5 | Toggle global dictation |
| Auto-paste dictation | ✓ | Pastes transcript at cursor after dictation |
| Whisper model | `ggml-base.en.bin` | Swap to `small`/`medium` for better quality |
| Ollama model | `llama3.1:8b` | Swap to `mistral:7b` for smaller, `qwen2.5:14b` for better |

### Swap the whisper model

```bash
# Smaller/faster
curl -L https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-tiny.en.bin \
  -o ~/Documents/MeetingNotes/models/ggml-tiny.en.bin

# Bigger/more accurate (1.5 GB, needs more RAM)
curl -L https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-medium.en.bin \
  -o ~/Documents/MeetingNotes/models/ggml-medium.en.bin
```

Then update Settings → Whisper.cpp → "GGML model path".

### Swap the Ollama model

```bash
ollama pull mistral:7b      # smaller
ollama pull qwen2.5:14b     # better notes, more RAM
```

Then update Settings → Ollama → Model.

## Troubleshooting

**"whisper-cli failed (3): failed to initialize whisper context" or "invalid model data (bad magic)"**
Your model file is corrupted or zero-byte (sometimes the initial download leaves a phantom-sized empty file). Re-download:

```bash
rm ~/Documents/MeetingNotes/models/ggml-base.en.bin
./scripts/setup.sh
```

**Summary fails with "Could not reach Ollama"**
The app now auto-starts Ollama on launch and before each summary. If it still fails:

```bash
brew services start ollama         # persistent across reboots
# OR
ollama serve                       # in a terminal
```

**Live transcript shows "tensor API disabled for pre-M5 and pre-A19 devices"**
That's an informational warning, not a real error, on M1/M2/M3/M4 hardware. If transcription still actually fails AND the model is valid, build a CPU-only whisper-cli as a fallback:

```bash
./scripts/build-whisper-cpu.sh
```

It writes `~/Documents/MeetingNotes/bin/whisper-cli`. Point Settings → Whisper.cpp → "whisper-cli binary" at that path.

**The app shows "Not Responding"**
Probably a stale TCC entry from a prior install. Reset and re-grant:

```bash
pkill -9 -x MeetingScribe
for svc in ScreenCapture Microphone Calendar AppleEvents Accessibility; do
  tccutil reset $svc com.tyleryannes.MeetingScribe
done
./install.sh
```

**System audio is silent in recordings**
You granted Screen Recording while the app was running. Quit and relaunch.

**Keychain keeps asking for password every time codesign runs**
Run once in Terminal so you can type your login password:

```bash
security set-key-partition-list \
    -S apple-tool:,apple:,codesign: \
    -s \
    ~/Library/Keychains/login.keychain-db
```
