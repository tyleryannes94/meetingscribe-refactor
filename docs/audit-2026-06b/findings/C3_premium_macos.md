# Industry/Competitive — Premium Native macOS Design Benchmarks
> Lens: what concretely makes Things 3, Craft, Linear, Notion Calendar, Arc, Raycast, and Apple's Tahoe-era apps read "clean and expensive" — chrome, materials, type, motion, command surfaces, settings, and icon-to-empty-state cohesion — and where MeetingScribe falls short of that bar.

## Benchmark research (June 2026 state)

- **macOS Tahoe / Liquid Glass is now the baseline for "native-feeling".** Tahoe sidebars and toolbars are translucent Liquid Glass that "reflect and refract what you're viewing, drawing more focus to your content"; the upcoming macOS 27 Golden Gate doubles down with uniform toolbars, edge-to-edge sidebars, and improved contrast. Apps that still draw opaque, fully-custom chrome now read as web ports. Sources: [Apple Support — What's new in macOS Tahoe 26](https://support.apple.com/en-us/122868), [MacRumors — Liquid Glass changes in Golden Gate](https://www.macrumors.com/2026/06/09/macos-golden-gate-liquid-glass/), [OpenMark — best native Tahoe apps](https://openmarkapp.com/blog/macos-tahoe-apps-liquid-glass), [Apple Newsroom — new software design](https://www.apple.com/newsroom/2025/06/apple-introduces-a-delightful-and-elegant-new-software-design/).
- **Linear's lesson: dim the chrome, light the content.** Their redesigns center on an "inverted L" of global chrome where the sidebar is *dimmed* so content takes precedence, labels/icons are pixel-aligned, and contrast is softened — polish you "feel after a few minutes." Their Liquid Glass piece is the best public spec of the premium-material stack: blur base + structural gradient + GPU specular highlight + lift-on-touch + variable edge blur on scroll + Increase Contrast fallback; they deliberately *skipped refraction* because it hurts dense pro UIs. Sources: [Linear — How we redesigned the Linear UI](https://linear.app/now/how-we-redesigned-the-linear-ui), [Linear — A calmer interface](https://linear.app/now/behind-the-latest-design-refresh), [Linear — A Linear spin on Liquid Glass](https://linear.app/now/linear-liquid-glass).
- **Things 3's lesson: whitespace, one typeface system, and one signature transition.** Reviews consistently attribute its premium feel to generous whitespace, system Dynamic Type compliance, and the open-a-todo animation where the row "smoothly transforms into a clear white piece of paper… fields tucked away until you need them." Sources: [Cultured Code — Things features](https://culturedcode.com/things/features/), [Things 3 review](https://medium.com/@smithtimmytim/review-things-3-for-mac-and-ios-114f4420f44b).
- **Raycast's lesson: the command surface is a non-activating floating panel measured in milliseconds.** Native AppKit, instant summon, and crucially: "interacting with the palette should not activate the owning app" — it floats above everything and returns you to where you were. Sources: [Raycast](https://www.raycast.com/), [Multi — nailing command palette activation behavior](https://multi.app/blog/nailing-the-activation-behavior-of-a-spotlight-raycast-like-command-palette).
- **Notion Calendar (Cron)'s lesson: keyboard-first + a menu bar surface that earns its pixels.** Single-key shortcuts, a command menu for everything, and a menu bar calendar showing upcoming events with one-tap "Join conferencing" / "Email participants" — the most-praised "small but mighty" details. Sources: [Notion Calendar keyboard shortcuts](https://www.notion.com/help/notion-calendar-keyboard-shortcuts), [Cron — menu bar settings](https://cronhq.notion.site/Menu-bar-settings-cb58322a14de49278bc2ecf24309b0ff), [Matthias Frank — Notion Calendar guide](https://matthiasfrank.de/en/notion-calendar/).
- **Arc's lesson: chrome earns trust by disappearing.** "Toolbars are hidden until needed… the interface emphasizes content rather than the chrome around it"; the command bar is the center of everything. Source: [Blake Crosley — Arc: reimagining the browser chrome](https://blakecrosley.com/guides/design/arc), [Arc review](https://dockshare.io/apps/arc).

## Full-app audit (through my lens)

**Strong (keep and amplify):**
- The NDS token discipline is genuinely good for an indie app: spacing scale, 3-speed motion constants + canonical spring (`UI/NotionDesign.swift:219-237`), semantic status/priority/due colors with non-color glyph redundancy (`NotionDesign.swift:247-279`), WCAG contrast helpers (`:287-307`), deterministic avatar gradients (`:177-194`). This is the raw material of an expensive app.
- The MS* button family has real press physics (spring scale + dim, `NotionDesign.swift:483-544`), and `bloomAmbientGlow()` (`UI/MSComponents.swift:27-35`) is a legitimate signature — a static, reduce-motion-safe brand light. Linear ships the same trick.
- Keep-alive tabs + warm caches (`UI/MainWindow.swift:94-110`) deliver the "instant" feel benchmarks are praised for.

**Weak / reads cheap (each falsifiable, with the premium replacement):**
1. **No app icon exists at all.** `Resources/` contains only `Entitlements.plist`, `Fonts/`, `Info.plist`; `Info.plist` has zero `Icon` keys and the Makefile copies no `.icns`. The app ships with the generic macOS icon — the single loudest "not a real product" signal, before a user sees one pixel of UI. (Tahoe additionally expects Icon Composer layered icons that tint/glass with the system.)
2. **Opaque, hand-rolled window chrome.** The shell is a plain `Window` (`MeetingScribeApp.swift:27`) wrapping a manual `HStack` of rail + content + chat (`MainWindow.swift:383-410`); the rail is a fixed 240pt opaque `NDS.sidebarBg` panel (`MainWindow.swift:178-180`, color defined `NotionDesign.swift:74`). No `NSVisualEffectView`, no sidebar material, no desktop show-through, not collapsible. Materials appear in exactly 4 places app-wide (`People/Graph/GraphFilterBar.swift:76`, `People/Graph/PeopleGraphView.swift:241`, `UI/MarkdownEditor.swift:821`, `UI/FloatingOverlay.swift:260`). On Tahoe, where Finder/Notes sidebars are Liquid Glass, this reads Electron.
3. **An in-rail Light/Dark toggle overriding the OS.** `AppearanceToggle` (`NotionDesign.swift:695-729`) sits permanently in the nav rail (`MainWindow.swift:151`) and drives `preferredColorScheme` (`MainWindow.swift:416`), defaulting everyone to dark (`:71`). No benchmark native app puts a theme switch in primary navigation; Things/Craft follow the system. It's the #1 web-app tell.
4. **⌘K is a search sheet, not a command surface.** `GlobalSearchView` is a fixed 620×480 opaque sheet (`UI/GlobalSearchView.swift:59-60`) presented via `.sheet` (`MainWindow.swift:468-475`) — it slides in like a document modal, contains zero verbs (search-only despite the "command palette" comment at `:4`), uses raw `.font(.callout)`/`.title3` instead of NDS type (`:82, :103`), and has no footer key-hints, no per-result actions. Raycast/Linear set the bar: floating non-activating panel, actions + entities, ⌘K-within-⌘K.
5. **First-run is off-brand.** `OnboardingSheet` is fixed 480×480 (`UI/OnboardingSheet.swift:48`), uses system `.title2` (`:62`), stock `.bordered`/`.borderedProminent` buttons (`:94-99, :113-114, :190-191`) — never `MSPrimaryButtonStyle` — gray step dots (`:203-211`), and a monospaced `~/path` as the hero content of screen one (`:78-87`). The brand (coral gradient, Bricolage display type, the waveform mark) is completely absent from the first five minutes.
6. **Settings is a 23-section infinite scroll.** One `Form` (`UI/SettingsView.swift:76`) with sections from "About" to "Whisper.cpp" to raw API-key fields (`:77-579`) in a fixed 560×580 window (`MeetingScribeApp.swift:145`). Whisper binary paths, MCP JSON config, and diagnostics sit beside everyday toggles. Premium apps ship a tabbed, toolbar-style settings window and bury plumbing under Advanced; settings design is where Mac users judge craftsmanship first.
7. **The menu bar extra wastes the highest-frequency surface.** `MenuBarView` (`UI/MenuBarView.swift:10-69`) is `.menuBarExtraStyle(.window)` rendered as a plain VStack of default buttons, caption text, and Dividers — zero NDS, no next-event card, no countdown. Cron/Notion Calendar's menu bar (next event + Join button) is the most-cited delight feature in that product.
8. **Chrome speaks two languages at once.** The toolbar uses stock bordered buttons tinted ad hoc (`MainWindow.swift:237`) above a fully custom rail; the rail repeats the wordmark while `.navigationTitle("MeetingScribe")` (`:417`) repeats it again in the title bar. Nav icons for Meetings (`person.2.fill`) and People (`person.2`) are nearly identical glyphs (`MainWindow.swift:22-28`) — a detail-level confusion Things/Linear would never ship.
9. **Motion tokens exist but hover/section animations bypass them.** Hard-coded `.easeOut(duration: 0.12)` at `NotionDesign.swift:601, :649` and `MainWindow.swift:685`; section switches are a flat opacity crossfade (`MainWindow.swift:96-104`) with no spatial direction; no `.contentTransition(.numericText())` anywhere for timers/counters; no signature detail-open transition (Things' "row becomes a sheet of paper").
10. **Empty states are system-gray.** `MSEmptyState` (`UI/MSComponents.swift:213-242`) renders `.secondary` SF Symbol + `.headline`/`.caption` system text — no display type, no brand tile, no gradient — so the moments a new user sees most are the least branded.

## Existing-plan items I rank highest

1. **Recents rail + ⌘K quick-switcher (2A)** — the command surface is the premium centerpiece; C3-2 below is the interaction spec it needs.
2. **Unify split panes under one `NavigationSplitView` shell (2A)** — prerequisite for native sidebar materials and free Tahoe chrome behavior (C3-3 builds on it).
3. **Gate all motion through `NDS.motion()` + 3-speed hierarchy (1F)** — tokens exist (`NotionDesign.swift:229-237`); adoption is the cheap half of a motion language.
4. **Loading skeleton tri-state (1F)** — perceived speed is most of "expensive"; Linear's calm-interface thesis exactly.
5. **Warm copy + photo hero avatar (2B)** — the Things-style emotional register the coach half needs.
6. **Dynamic Type round 2 on fixed-size sheets (2H)** — Things 3's system-text-size compliance is repeatedly cited as premium craft; the 480×480 onboarding clips today.

## NET-NEW recommendations

### C3-1 — Ship a real app icon + one brand-mark pipeline (icon → onboarding → empty states)
- **What/why:** There is **no icon**: `Resources/Info.plist` has no icon key and no `.icns`/asset catalog exists in the repo. Design one mark — the existing coral→lilac waveform tile (`MainWindow.swift:117-124`) is already it — and produce a Tahoe layered icon via Apple's Icon Composer (so it participates in system tint/glass modes per [Apple's 2025 design resources](https://appleinsider.com/articles/25/06/11/apple-design-resources-update-helps-developer-make-liquid-glass-apps-for-ios-26-macos-tahoe)), wire `CFBundleIconFile` + Makefile copy, then reuse the same mark as the onboarding hero and the `MSEmptyState` glyph tile. Benchmark: every app in this cohort is recognizable from its icon alone.
- **User value:** Dock, Cmd-Tab, Spotlight, notifications, and the menu bar all stop screaming "unfinished side project."
- **Effort:** S–M · **Impact:** High · **Depends on:** none

### C3-2 — Rebuild ⌘K as a Raycast-grade floating command palette (verbs + entities)
- **What/why:** Replace the `.sheet`-presented search (`MainWindow.swift:468-475`, `GlobalSearchView.swift:59`) with a centered, non-activating floating `NSPanel` over `.regularMaterial`/glass that fades+scales in (≈120ms). Two result classes: **commands** ("Start recording", "New task", "Log encounter with <person>", "Open Settings") and **entities** (people/meetings/tasks/notes), recents-first on empty query, fuzzy-matched, with a footer hint bar (`↑↓ select · ↩ open · ⌘↩ secondary · ⌘K actions`) and a per-result actions submenu (Raycast's ⌘K-inside-⌘K). This is the premium spec for the plan's "Recents + Cmd-K" line item — the plan says *what*, this is *how it must feel*. Benchmarks: [Raycast](https://www.raycast.com/), [Multi's activation-behavior writeup](https://multi.app/blog/nailing-the-activation-behavior-of-a-spotlight-raycast-like-command-palette), Linear's ⌘K.
- **User value:** Every action and entity is ≤2 keystrokes away; the single feature reviewers use to label an app "Linear-class."
- **Effort:** M · **Impact:** High · **Depends on:** 2A router/EntityLink

### C3-3 — Native chrome & materials pass: translucent sidebar, hidden title bar, glass floating surfaces
- **What/why:** Replace the opaque rail background (`MainWindow.swift:180`, `NotionDesign.swift:74`) with an `NSVisualEffectView(.sidebar, blendingMode: .behindWindow)` wash tinted by the plum palette at low alpha so the desktop shows through; hide the redundant title (`MainWindow.swift:417`) and run content full-height under a transparent titlebar; render `MeetingRecordDock`, `FloatingOverlay`, and the toast layer on Tahoe `glassEffect`-style material (fallback `.ultraThinMaterial` on macOS 14/15), following Linear's recipe — blur base + structural gradient + lift-on-press — and Linear's restraint (no refraction over dense text). Dim the rail relative to content per Linear's inverted-L principle. Sources: [Apple — macOS Tahoe](https://support.apple.com/en-us/122868), [Linear — Liquid Glass](https://linear.app/now/linear-liquid-glass), [Linear — UI redesign](https://linear.app/now/how-we-redesigned-the-linear-ui).
- **User value:** The app stops looking like a themed web view and starts looking like it belongs on a 2026 Mac — the largest single "expensive" delta available.
- **Effort:** M–L · **Impact:** High · **Depends on:** plays best after 2A's `NavigationSplitView` unification

### C3-4 — Kill the in-rail Light/Dark toggle; follow the system
- **What/why:** Delete `AppearanceToggle` from the rail (`MainWindow.swift:151`), default `preferredColorScheme` to system, and move an Auto/Light/Dark picker into Settings → Appearance. No benchmark app (Things, Craft, Notion Calendar, Raycast main window) puts theming in primary nav; the current dark-by-default override (`MainWindow.swift:71`) also fights Tahoe's auto-appearance. Reclaims rail space for the ⌘K affordance.
- **User value:** One less web-ism; the app respects the Mac it runs on.
- **Effort:** S · **Impact:** Med · **Depends on:** light palette spot-check (NDS light values exist, `NotionDesign.swift:73-90`)

### C3-5 — Tabbed, toolbar-style Settings with an Advanced basement
- **What/why:** Split the 23-section scroll (`SettingsView.swift:76-579`) into a native settings `TabView` (toolbar style): **General** (you, storage, appearance) · **Recording** (capture, calendar, impromptu) · **AI** (Whisper, Ollama, dictation) · **People** · **Integrations** (MCP, Notion, Linear, Drive, Obsidian, phone) · **Advanced** (binary paths, diagnostics, metrics, version/update). Use `.formStyle(.grouped)`, free the fixed 560×580 frame (`MeetingScribeApp.swift:145`). Settings design is a canonical craft signal — Things 3's five-tab settings vs. this scroll is the cheapest credibility gap to close.
- **User value:** Findability (23 sections → 6 tabs), and API keys/binaries stop sitting next to "Daily brief."
- **Effort:** M · **Impact:** Med-High · **Depends on:** none

### C3-6 — Menu bar extra rebuilt to the Cron standard (next-event card + live recording state)
- **What/why:** Redesign `MenuBarView` (`UI/MenuBarView.swift:10-69`) as a branded card surface: NDS type, the **next event as a hero card** with countdown and one-click "Join & Record", a compact drifting-people strip, and while recording, an elapsed timer with `.contentTransition(.numericText())` plus a timer in the menu bar icon itself. Today it's default-styled buttons and captions — the only surface most users see hourly, and the least designed. Benchmark: [Cron/Notion Calendar menu bar](https://cronhq.notion.site/Menu-bar-settings-cb58322a14de49278bc2ecf24309b0ff).
- **User value:** The highest-frequency touchpoint becomes the best ad for the app; join+record drops to one click from anywhere.
- **Effort:** M · **Impact:** High · **Depends on:** none

### C3-7 — Typography re-tiering: SF Pro for UI, Bricolage for display moments only
- **What/why:** Two bundled web fonts for *everything* (`NotionDesign.swift:112-113`) is the deepest "not native" tell — and it's already inconsistent: onboarding, settings, search, and the menu bar all use raw system styles (`OnboardingSheet.swift:62`, `SettingsView.swift:74`, `GlobalSearchView.swift:103`, `MenuBarView.swift:30`). Re-tier: **SF Pro (system) for all body/UI text** (free optical sizing, full Dynamic Type, Tahoe rendering), keep **Bricolage Grotesque exclusively for display** (page titles, onboarding hero, paywall, empty-state titles), retire Plus Jakarta Sans. Monospaced digits for all timers/counts. Things 3's praised type system is exactly this: system text + one personality face. Sources: [Apple HIG typography](https://developers.apple.com/design/human-interface-guidelines/macos/visual-design/typography/), [Cultured Code](https://culturedcode.com/things/features/).
- **User value:** Native rendering/legibility everywhere, brand voice concentrated where it lands, and the inconsistency disappears by construction.
- **Effort:** M (mechanical sweep; NDS.font already centralizes) · **Impact:** High · **Depends on:** none

### C3-8 — Motion language v2: one signature transition + numeric content transitions
- **What/why:** Beyond the plan's "gate motion" item, define the *vocabulary*: (a) a signature detail-open — list row scales/morphs into the detail pane header (Things' todo→paper moment) using `matchedGeometryEffect` on title+avatar; (b) directional section changes (subtle 8pt slide + fade keyed to nav order) replacing the flat crossfade (`MainWindow.swift:96-104`); (c) `.contentTransition(.numericText())` on recording timers, health scores, task counts; (d) `symbolEffect(.bounce)` on record start/stop; (e) migrate the hard-coded hover `easeOut(0.12)`s (`NotionDesign.swift:601, :649`, `MainWindow.swift:685`) onto `NDS.motionFast`. All reduce-motion-gated. Benchmarks: Things 3's open animation, Linear's lift-on-touch physics ([Linear](https://linear.app/now/linear-liquid-glass)).
- **User value:** Motion becomes a brand asset instead of incidental; "smooth" is the word reviews reach for.
- **Effort:** M · **Impact:** Med-High · **Depends on:** 1F motion gating

### C3-9 — Brandize the first five minutes: onboarding + empty states on NDS
- **What/why:** Rebuild `OnboardingSheet` on the design system: brand-mark hero (the C3-1 icon, animated once), Bricolage display titles, `MSPrimaryButtonStyle` CTAs replacing `.borderedProminent` (`OnboardingSheet.swift:113, :190`), lilac progress dots, friendly location copy ("Keep everything in iCloud Drive" with the path as secondary detail, not monospaced hero `:78-87`); upgrade `MSEmptyState` (`MSComponents.swift:227-241`) with a 56pt avatar-gradient glyph tile, display-type title, NDS body, and exactly one `MSPrimary` CTA. Distinct from the held 1E copy sweep (that's voice); this is the visual system. Benchmark: Craft/Arc onboarding, Things 3 empty states.
- **User value:** Day-0 visual quality matches the marketing promise; empty states sell instead of apologize.
- **Effort:** M · **Impact:** High · **Depends on:** C3-1 (mark), pairs with held item 1E
 
### C3-10 — Chrome de-duplication: one wordmark, distinct glyphs, one toolbar language
- **What/why:** Hide the title-bar duplicate of the wordmark (`MainWindow.swift:417` + rail brand row `:117-129` say "MeetingScribe" twice, three times while onboarding shows); give Meetings a calendar-family glyph so it stops twinning People (`person.2.fill` vs `person.2`, `MainWindow.swift:22-28`); restyle toolbar items as quiet `NotionIconButton`-class controls with **one** filled primary per page (the ToolbarModel primary), removing the ad-hoc `.tint()` mixing (`MainWindow.swift:237`). Arc's principle: chrome earns trust by disappearing ([Crosley on Arc](https://blakecrosley.com/guides/design/arc)); Linear's: align and dim the inverted-L ([Linear](https://linear.app/now/how-we-redesigned-the-linear-ui)).
- **User value:** Calmer window, faster glyph scanning, a single visual hierarchy from rail to toolbar.
- **Effort:** S · **Impact:** Med · **Depends on:** none

### C3-11 — Single-key, keyboard-first layer (beyond ⌘K)
- **What/why:** Notion Calendar's most-praised trait is that *unmodified* keys do work (`S` to schedule, `T` for today). Add a context-key layer: in lists, `J/K` or arrows move selection with a visible focus ring, `↩` opens, `E` edits, `D` toggles done (Tasks), `R` starts recording from Today, `/` focuses search — surfaced via a `?` overlay cheat-sheet. The app already has ⌘-shortcuts (`MeetingScribeApp.swift:89-126`) but nothing unmodified. Sources: [Notion Calendar shortcuts](https://www.notion.com/help/notion-calendar-keyboard-shortcuts), [Cron global shortcuts](https://cronhq.notion.site/Global-keyboard-shortcuts-e933a55e7fb648028b09cedf933d3e76).
- **User value:** Power-user speed (triage a task list with zero mouse) and the "this team sweats details" reputation that drives word of mouth.
- **Effort:** M · **Impact:** Med · **Depends on:** 2A focus/selection unification

## Top 3 picks

1. **C3-3 Native chrome & materials pass** — the single highest-leverage "expensive" move; everything else sits on this stage. **This is also my single highest-priority recommendation overall.**
2. **C3-1 App icon + brand-mark pipeline** — the app currently has *no icon*; cheapest fix with the largest perceived-quality delta per hour.
3. **C3-2 Raycast-grade ⌘K command palette** — converts the planned quick-switcher into the marquee interaction that defines "Linear-class" in 2026.
